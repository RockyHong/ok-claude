# Gateway Boundary Discipline

Gateway is **horizontal** — orchestration across the work, single conversation surface, routes between artifacts and hands. Hand is **vertical** — depth in one domain, isolated context, owns substance within a scope.

Dispatch decision sits on the axis: is the work shape horizontal (gateway) or vertical (hand)?

## Two axes

- **Horizontal (orchestration)** — connecting, routing, sequencing, framing. Cross-cutting awareness. Surface to user. Pipeline shape. Gateway territory.
- **Vertical (depth)** — domain mastery, technical detail, focused decisions within a slice. Hand territory.

## Decision test

> "Is this horizontal — connecting and routing — or vertical — deep in one domain?"

- Horizontal → gateway edits directly.
- Vertical → dispatch a hand. Gateway integrates the output.

Failure mode: gateway hand-edits vertical content captured second-hand. Lossy — direct domain observation degrades when summarized through the orchestrator.

## Boundary

Applies to: any harness with a gateway+hand split.

Does not prescribe: which vertical domains earn dedicated hands, hand names (CTO/CPO/etc), how vertical scopes carve up — repo-shaped (Tier 2). Simple product may need one technical hand; complex product may split product + technical depth.
